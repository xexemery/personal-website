2025-08-08 // Game jam updates!

Hello! It's been a few weeks since I've posted an update, but I've been busy participating in lots of game jams! I've been enjoying it a lot, although it has been tiring. I have been using Godot for everything so far, and I feel like I learn so many new things each time. I'd like to eventually add a page on this site to show all the games I've made, and add some devlogs to show the progress. But for now I'll do a brief recap:

// Kenney jam

First, I participated in Kenney Jam 2025, where I made my very first game! This was a 48-hour jam which felt like a good place to start, plus the requirement to only use assets from Kenney or KayKit meant I didn't need to worry about making my own art yet. I actually ended up using Kenney assets for all three jams I've done so far, as it means I can focus on the gameplay and learning how to use the engine in different ways.

The theme was POWER, so I made a small puzzle game about turning lights on and off. It's called Lights On! and you can play it here! I'm overall quite pleased with this one, especially considering it was the first game I ever made. I got some good feedback on it as well. I really like the title art I made for it too!

The main thing I struggled with here was puzzle design. I ended up having to rush through designing the levels on the final day, because I had to spend so long bug fixing to get the main mechanic working properly! There are still a couple of small bugs in the final build, so I'd like to go back and fix those at some point. I reckon with the additional experience and knowledge I've gained, I'll be able to rework the code a bit as well, to avoid the problems I was having with checking the win condition for each level at the right time.

I also intended to have more platforming mechanics in the game, but again, time was short, and it was really difficult to think up puzzle ideas under so much time pressure! I have been finding it hard not having much time to just sit and think about level design during these game jams so far, but it does make me very keen to try and work on some projects without the time pressure, as I think I could make something really good!

// Melon jam

Now, I intended to spend some time progressing The Odin Project in between the Kenney jam and GMTK a couple weeks later, but I found it just wasn't what I wanted to focus on. I wanted to continue working on game dev, as I felt I had only just scratched the surface! So, I ended up participating in Melon Jam 6 a week later! This was a much smaller jam, and this time was 72 hours long, so it felt like a good mid-point between the Kenney and GMTK jams.

The theme for this one was COMPOUND, which I found was a lot harder to think of ideas for. I ended up making a game called Jeremiah's Compound that incorporates the theme in a few different ways. This was a very different type of game to Lights On! and I actually found it quite hard to make something decent within the time limit. I feel like if I had made it without time constraints, I would've added a lot more to the combat and dialogue systems.

But, very importantly, through making this game, I learnt how to make a basic text box and UI! That was honestly the most valuable thing I gained from making this one. I also just felt way more confident using Godot this time around, and felt like I was getting the hang of things a lot more.

// GMTK jam

Next up was the big one, the GMTK Game Jam 2025. By this point, my partner (who has also been participating along with me) and I were both honestly feeling quite exhausted! We decided to take this one a lot slower. Ideally, we might have worked together for the full four days to make something amazing, but that just didn't seem like a good idea with how we were feeling.

The theme for GMTK was LOOP, which due to being so broad, honestly made it really hard to come up with ideas. There were a lot of things I could've done, if I'd had more energy to dive in and learn how to implement them, but I wanted to keep it simple this time, because four days of doing pure game dev and not much else didn't feel sustainable.

In the end, I kept thinking about this short film by Max Zavidow whenever I tried to come up with ideas for the theme. So I decided to try and adapt it into a game. And I was mostly successful! You can play the game, titled Tricycle No More, here.

This one involved a lot of cutscenes, something I had no prior experience with. I ended up spending very minimal time on the levels and gameplay, because it felt more important to get the narrative portrayed well. I also just didn't feel like spending ages designing platformer levels under the time constraints. To get around this, I ended up adding a timer to each level, so it was more of a speedrunning challenge. That way, the levels could be really simple as the focus would be on jumping to the end as fast as possible.

Through making the cutscenes, I learnt so much about using animations in Godot! I'm really proud of the opening scene especially. That said, I ended up using a lot of await statements in my code, and I feel like there must be a better way to do that. That's something I'll look into for future projects. But overall, I am pleased with the outcome, even if the gameplay is a bit janky.

// Where I'm at now

So, this brings us to the present, and I am actually participating in another jam! I had signed up to a few more, before I ended up getting thoroughly exhausted from the past few weeks. Fortunately, they are all over a longer time period, so I'm taking it slow and just trying to make something small that I can learn from within the time frame.

Currently, I am making a game for LOWREZJAM 2025! The challenge for this one is to make a game using 64x64 resolution or less. I heard about this one after the Kenney jam, and it really appealed to me as a pixel art enjoyer. There are ten optional themes, and I was inspired by a few different ones to make a simple fishing game.

I am really enjoying the process so far, as it's very different to what I've made before, and I also don't need to spend time on level design! I'm also creating my own assets this time around! During GMTK, I ended up using PICO-8 to add a few extra sprites to the Kenney pack I was using, and it was really simple and enjoyable to do! I considered entirely developing the game in PICO-8, but since I have no experience with it, I decided it would be best to just focus on learning Godot for now. But it has been really great to use as a sprite editor!

// What's next

Coming up next week is the B1T Jam, which is a 7-day long jam to make a game using only 2 colours! There are also some additional incentives to use an open source JavaScript library called KAPLAY, and since I need to keep working on my JavaScript skills for my job search, I think this could be a fun way to do that!

It wasn't too hard at all, so now I have plenty of options for creating my own art assets going forward.

Finally, the Brackeys Game Jam takes place later this month, a week-long jam hosted by Brackeys, whose tutorials I used to learn Godot when I was just getting started. I feel like taking part in this jam will be a great full circle moment!
